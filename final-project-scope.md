# Project Scope Statement

## Project Name

Project 03 Final Project – Personal Portfolio Website

## Project Purpose and Justification

The purpose of this project is to build a personal portfolio website that brings together my professional background, my education, and some of the interests that are important to me. I want the site to work as both a class project and something I can keep building on later. Since I already work in a technical and public-facing role, I think it makes sense to create a website that shows both the professional side of my background and the skills I am building through the Computing Applications program. This project also fits the goals of the course because it uses web design, content organization, and project planning all in one final product.

## Project Objectives

The main objective of this project is to create a clean, organized website that introduces who I am and what I do. I want the site to include my work background, current degree program, technical interests, and a few personal interests so it feels like an actual portfolio and not just a class exercise. Another goal is to improve on the earlier assignments by making the final site look more polished, more complete, and better organized overall.

## High-Level Requirements

The final website will need to include the following:

* A homepage or landing page
* A short bio or introduction section
* Information about my current degree program and educational background
* A section about my work experience and career interests
* A section covering personal interests such as photography, GIS, anime, or outdoor interests
* Links to GitHub and course-related work
* Consistent styling using CSS
* Clear navigation between sections or pages
* A live published version of the site through GitHub Pages

## Product Description / Final Deliverable

The final deliverable will be a completed personal portfolio website published through GitHub Pages. The website will be built using HTML and CSS and will serve as a central place to present my background, interests, and course work. The site should feel complete enough to stand on its own as a portfolio website, while still meeting the requirements of Project 03 for this class.

## Project Boundaries

### In Scope

The following work is included in this project:

* Building the structure and layout of the website
* Writing the content for the different sections of the site
* Styling the website with CSS
* Adding navigation, images if needed, and outside links
* Publishing the finished site to GitHub Pages
* Revising earlier work so the final site feels more complete and polished

### Out of Scope

The following work is not included in this project:

* Building a business website or online store
* Creating advanced JavaScript features or database-driven content
* Building a custom web application
* Creating a full photography portfolio with large image galleries
* Building a GIS web app or any advanced interactive map tools

## Constraints

The biggest constraint is time. This project has to be completed during the semester while balancing work and other assignments, so I need to keep the scope realistic. Another constraint is that the project needs to stay within the tools and skills we have been using in class, mainly HTML, CSS, Markdown, GitHub, and GitHub Pages. I also need to make sure the project stays focused enough that I can finish it well instead of trying to include too many ideas and ending up with something unfinished.

## Assumptions

This project assumes that I will continue using GitHub Pages to host the site and that I can build from the work I have already done in earlier assignments instead of starting over completely. It also assumes that the website content will come from my own background, work experience, school experience, and personal interests. I am also assuming that the final project is supposed to show growth from the earlier work in the class, so I plan to use that earlier work as the base and improve it rather than making something unrelated.

## Risks

One risk is making the project too big by trying to include every possible interest or idea I have. Another risk is spending too much time changing colors, layout, or small design details instead of finishing the content and structure first. There is also a risk of technical issues with GitHub Pages, broken links, or file organization if I do not keep the project folders and files clean. The best way to manage those risks is to keep the site focused and build the main content first before worrying too much about extra features.

## Major Milestones

1. Review the full Project 03 assignment requirements
2. Decide on the final layout and sections for the website
3. Complete the project scope statement
4. Build or revise the HTML pages for the site
5. Apply CSS styling and improve the visual layout
6. Add the final written content, links, and images
7. Test the site and fix layout or navigation problems
8. Publish the final version to GitHub Pages and submit the required links

## Acceptance Criteria

The project will be complete when:

* The website is published and accessible online
* The site includes the major sections described in this scope statement
* The navigation works correctly
* The content is complete and organized
* The visual design is consistent across the site
* The project meets the requirements for Project 03
* The GitHub repository and live website link are ready to submit

## Approval / Scope Commitment

This scope statement is the plan for my final project. It defines what I intend to build, what I am leaving out, and the general direction of the website. My goal is to use it to keep the project realistic, organized, and focused so I can finish a strong final product without letting the scope get out of hand.
