# Project Change Request

## Proposed Change

Add an embedded Google Map to my Midterm Project website that shows my hometown. The map would be added to the homepage or About section so visitors can quickly see where I’m from without having to leave the site.

## Reasons for the Change

I can see why a stakeholder would want this added because it gives the website a little more personality and context. Since the project is a personal website, showing my hometown on a map helps connect the information on the page to a real place. It also gives visitors a better sense of who I am and where my background is rooted.

## How This Change Will Add Value

This change would add value by making the website feel more personal and more interactive. Instead of only listing my hometown in text, the map gives visitors a visual reference that is quicker and easier to understand. It also helps the site feel more complete and polished. For a personal website, small features like this can make the page more engaging without changing the overall purpose of the project.

## How This Change Will Affect the Constraints

### Time

The impact on time would probably be pretty small. Embedding a Google Map is not a major feature, but it would still take some time to add it, place it correctly on the page, and test it to make sure it displays properly.

### Quality

If done well, this change would improve the overall quality of the website because it adds a visual element and gives the page more depth. At the same time, I would still need to make sure it doesn’t throw off the layout or look awkward on mobile devices. So while it could improve the site, it would still need a little testing to make sure it fits cleanly with the rest of the design.

### Budget

The budget impact would basically be none for this project. Since this is a school website and the map would just be embedded from Google Maps, there would not be any real financial cost. The only cost would be the extra time spent adding and checking the feature.
