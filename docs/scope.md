# Project Scope Statement

## Project Name

Project 03 Final Project – Personal Portfolio Website

## Project Purpose and Justification

The purpose of this project is to build a personal portfolio website that brings together my professional background, my education, and some of the interests that are important to me. I want the website to serve as both a class project and something I can continue to build after the course is complete. Since I work in a technical and public-facing profession while pursuing a degree in Computing Applications, this project provides an opportunity to showcase both my professional experience and the technical skills I have developed throughout the semester.

## Project Objectives

The primary objective of this project is to create a professional portfolio website that introduces who I am, highlights my educational background, and summarizes my work experience and career goals. The website should demonstrate the HTML and CSS skills learned during the course while providing a clean, organized, and easy-to-navigate design.

Additional objectives include:

- Creating a professional online presence.
- Demonstrating proper HTML and CSS structure.
- Publishing the completed website using GitHub Pages.
- Documenting the project using project management principles learned during the course.

## High-Level Requirements

The project will include the following:

- A homepage introducing who I am.
- An About page describing my background and interests.
- Navigation between all pages.
- A consistent design using a single external CSS stylesheet.
- Documentation stored within a `docs` folder.
- Publication through GitHub Pages.
- A public GitHub repository containing all project files.

## Product Description / Final Deliverable

The final deliverable will be a completed two-page personal portfolio website hosted on GitHub Pages. The repository will also include the required project documentation, including the Project Scope Statement, Project Plan, and Project Retrospective. Together these documents will demonstrate both technical and project management skills developed throughout the course.

## Project Boundaries

### In Scope

The following work is included in this project:

- Creating a personal portfolio website using HTML and CSS.
- Creating a homepage and About page.
- Developing a consistent visual style.
- Publishing the website using GitHub Pages.
- Creating all required project documentation.
- Organizing the repository according to the project requirements.

### Out of Scope

The following work is not included in this project:

- JavaScript programming.
- Database integration.
- User login systems.
- E-commerce functionality.
- Interactive GIS applications.
- Advanced web frameworks.

## Constraints

The primary constraint for this project is time. The project must be completed while balancing full-time employment and other coursework. Another constraint is limiting development to HTML, CSS, GitHub, GitHub Pages, and Markdown, since those are the primary technologies covered in this course.

## Assumptions

This project assumes that GitHub Pages will continue to provide hosting for the website and that all required tools will remain available throughout the project. It also assumes that the website content will be based on my own educational background, professional experience, and personal interests.

## Risks

Potential risks include:

- Running out of time before the submission deadline.
- Repository organization mistakes.
- Broken hyperlinks between pages.
- GitHub Pages publishing issues.
- Spending too much time making visual improvements instead of completing required deliverables.

These risks will be managed by completing the required documentation first, testing all links before submission, and keeping the project focused on the assignment requirements.

## Major Milestones

1. Review project requirements.
2. Complete the Project Scope Statement.
3. Create the GitHub repository.
4. Develop the homepage.
5. Develop the About page.
6. Apply CSS styling.
7. Complete the project documentation.
8. Test navigation and website functionality.
9. Publish the website using GitHub Pages.
10. Submit the GitHub repository.

## Acceptance Criteria

The project will be considered complete when:

- The GitHub repository is public.
- The website is successfully published using GitHub Pages.
- The repository contains the required documentation.
- Navigation between pages functions correctly.
- The website uses a single external CSS stylesheet.
- The project satisfies all Project 03 assignment requirements.

## Approval / Scope Commitment

This scope statement defines the overall direction and boundaries of the project. It establishes what will and will not be included while providing a clear plan for completing the final portfolio website. The goal is to deliver a professional, organized, and functional website that demonstrates both technical knowledge and effective project management practices developed throughout this course.
