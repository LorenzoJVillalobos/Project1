# Project Plan

## Work Breakdown Structure

1. Define project scope
2. Create repository
3. Build homepage
4. Build About page
5. Apply CSS styling
6. Test navigation and links
7. Publish to GitHub Pages
8. Submit project

## Risk Analysis (TAME)

| Risk | TAME Strategy |
|------|---------------|
| Running out of time | Treat by focusing on required features first |
| Broken links | Avoid by testing every page before submission |
| GitHub Pages issues | Mitigate by publishing early and testing frequently |
| Scope becoming too large | Eliminate by limiting the project to HTML and CSS only |

## Schedule

- Week 1: Planning and scope
- Week 2: Website development
- Week 3: Testing and documentation
- Week 4: Final revisions and submission
