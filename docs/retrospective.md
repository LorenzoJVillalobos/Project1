# Project Retrospective

## What went well?

The project planning process helped me stay organized throughout development. Using GitHub and GitHub Pages allowed me to track changes and publish my work online. Building the website improved my HTML and CSS skills and gave me a better understanding of project organization.

## What went wrong?

The biggest challenge was organizing the repository correctly. I accidentally created folders with names like index.html instead of creating files, which caused confusion and required additional cleanup. I also spent more time than expected learning GitHub and GitHub Pages.

## What would I do differently?

If I were starting again, I would plan the repository structure before uploading files. I would also publish the website earlier so I could identify problems sooner. Overall, I would spend less time reorganizing files and more time improving the website itself.
